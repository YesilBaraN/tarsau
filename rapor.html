1. Giriş

Bu projede, Linux ortamında C dilinde çalışan tarsau adında bir arşivleme programı geliştirilmiştir. Program, tar, rar veya zip benzeri şekilde çalışmakta ancak herhangi bir sıkıştırma işlemi yapmamaktadır. Temel olarak birden fazla metin dosyasını tek bir .sau uzantılı arşiv dosyasında birleştirmekte ve bu arşiv dosyasından tekrar orijinal dosyaları çıkartabilmektedir.

Program komut satırından iki farklı modda çalışmaktadır:

  • tarsau -b dosya1 dosya2 ... -o arsiv.sau → Dosyaları birleştirip arşiv oluşturma
  • tarsau -a arsiv.sau [dizin] → Arşiv dosyasını açma

2. Programın Genel Yapısı

Proje tek bir C kaynak dosyasından (tarsau.c) ve bir Makefile dosyasından oluşmaktadır. Program içerisinde kullanılan temel fonksiyonlar ve görevleri aşağıdaki tabloda özetlenmiştir:

Fonksiyon Görevi
metin_mi() Bir dosyanın ASCII metin dosyası olup olmadığını kontrol eder
dosya_boyu() stat() ile dosyanın boyutunu bayt cinsinden döndürür
izinleri_al() Dosyanın izinlerini octal string olarak okur (ör: "0644")
arsiv_olustur() -b modunda dosyaları birleştirip .sau arşivi oluşturur
arsiv_ac() -a modunda arşiv dosyasını açıp dosyaları çıkartır
kullanim_bilgisi() Yanlış kullanımda yardım mesajı gösterir
main() Komut satırı argümanlarını ayrıştırır ve ilgili fonksiyonu çağırır

Kullanılan kütüphaneler:

3. Geliştirme Süreci

a) Arşiv Dosya Formatı (.sau)

Arşiv dosyası iki ana bölümden oluşmaktadır. İlk bölüm organizasyon (içindekiler) bilgilerini, ikinci bölüm ise arşivlenmiş dosyaların içeriklerini barındırmaktadır. Format şu şekildedir: 

[10 bayt: org bölüm boyutu][|dosya1_adi,izin,boyut||dosya2_adi,izin,boyut|...][dosya1_icerigi][dosya2_icerigi]...

İlk 10 bayt, organizasyon bölümünün ASCII formatındaki sayısal boyutunu tutar. Bu sayede arşiv açılırken önce bu 10 bayt okunarak organizasyon bölümünün ne kadar yer kapladığı anlaşılır. Her dosya kaydı |dosya_adi,izinler,boyut| formatında pipe karakterleriyle çevrelenir. Dosya içerikleri ise organizasyon bölümünün hemen ardından herhangi bir ayırıcı olmadan art arda yerleştirilir.

Örnek bir arşiv dosyasının iç yapısı:

0000000030|t1,0755,6||t2,0644,7|merhababurada!
|________| |__________________|  |___________|
  10 bayt    org bölümü (30b)     dosya içerikleri

b) Metin Dosyası Kontrolü

Programın sadece metin dosyalarını arşivlemesi gerektiği için, her giriş dosyası öncelikle ASCII kontrolünden geçirilir. Dosyadaki her bayt tek tek okunur ve 0-127 aralığında olup olmadığı kontrol edilir. Ayrıca tab, newline ve carriage return dışındaki kontrol karakterleri de reddedilir. Bu sayede binary dosyalar arşivlenmez. 

int metin_mi(const char *yol)
{
    FILE *f = fopen(yol, "rb");
    if (f == NULL)
        return 0;

    int c;
    while ((c = fgetc(f)) != EOF) {
        if (c > 127) {            /* ascii disinda */
            fclose(f);
            return 0;
        }
        /* tab newline cr disindaki kontrol karakterlerini reddet */
        if (c < 32 && c != '\t' && c != '\n' && c != '\r') {
            fclose(f);
            return 0;
        }
    }
    fclose(f);
    return 1;
}

c) Arşiv Oluşturma (-b)

arsiv_olustur() fonksiyonu aşağıdaki adımları sırasıyla gerçekleştirir:

  1. Dosya sayısının 32'yi aşmadığı kontrol edilir
  2. Her dosyanın var olduğu ve okunabildiği doğrulanır
  3. Her dosyanın metin dosyası olup olmadığı metin_mi() ile kontrol edilir
  4. Toplam boyutun 200 MB'ı geçmediği hesaplanır
  5. Her dosya için |dosya_adi,izin,boyut| formatında organizasyon kaydı oluşturulur
  6. Çıktı dosyasına sırasıyla 10 baytlık başlık, organizasyon bölümü ve dosya içerikleri yazılır

Fonksiyonun organizasyon bölümünü oluşturan kısmı:

/* organizasyon bolumunu hazirla */
char org[8192];
org[0] = '\0';

for (i = 0; i < adet; i++) {
    char izin[8];
    izinleri_al(liste[i], izin);
    long b = dosya_boyu(liste[i]);

    char parca[512];
    sprintf(parca, "|%s,%s,%ld|", liste[i], izin, b);
    strcat(org, parca);
}

int org_boy = (int)strlen(org);

Dosyaya yazma işlemi:

/* baslik: 10 baytlik org boyutu */
fprintf(af, "%010d", org_boy);

/* organizasyon kayitlari */
fputs(org, af);

/* dosya iceriklerini siraya koy */
for (i = 0; i < adet; i++) {
    FILE *gf = fopen(liste[i], "r");
    int ch;
    while ((ch = fgetc(gf)) != EOF)
        fputc(ch, af);
    fclose(gf);
}

d) Arşiv Açma (-a)

arsiv_ac() fonksiyonu tam tersi işlemi yapar:

  1. Arşiv dosyasının ilk 10 baytı okunarak organizasyon bölümünün boyutu öğrenilir
  2. Organizasyon bölümü okunup her kayıt parse edilerek dosya adı, izin ve boyut bilgileri çıkartılır
  3. Hedef dizin belirtilmişse oluşturulur
  4. Her dosya için boyut bilgisi kadar veri okunarak hedef dizine yazılır
  5. chmod() ile orijinal dosya izinleri geri yüklenir

Kayıt ayrıştırma ve dosya çıkartma kısmı:

/* alanlari ayir:  ad,izin,boyut */
char ad[256], izin_s[16];
long boy;
sscanf(kayit, "%[^,],%[^,],%ld", ad, izin_s, &boy);

/* icerigi oku */
char *veri = (char *)malloc(boy + 1);
fread(veri, 1, boy, af);

/* dosyayi diske yaz */
FILE *df = fopen(yol, "w");
fwrite(veri, 1, boy, df);
fclose(df);

/* izinleri geri yukle */
int mod = (int)strtol(izin_s, NULL, 8);
chmod(yol, mod);

4. Kod Açıklamaları

Sabitler

Programın başında tanımlanan sabitler, ödevin gereksinimlerine göre belirlenmiştir:

#define EN_FAZLA_DOSYA   32                    // max dosya sayisi
#define BOYUT_SINIRI     (200 * 1024 * 1024)   // 200 MB
#define BASLIK_UZUNLUK   10                    // org boyut alani
#define ONAYLANMIS_UZANTI ".sau"               // arsiv uzantisi
#define STANDART_CIKTI    "a.sau"              // varsayilan cikti

İzin Okuma

Dosya izinleri stat() sistem çağrısı ile okunmaktadır. st_mode alanından sadece izin bitleri (& 0777) alınarak octal formatta string olarak saklanır. Bu format sayesinde arşiv açılırken strtol() ile tekrar sayısal değere çevrilip chmod() fonksiyonuna verilir. 

void izinleri_al(const char *yol, char *sonuc)
{
    struct stat bilgi;
    if (stat(yol, &bilgi) != 0) {
        strcpy(sonuc, "0644");
        return;
    }
    sprintf(sonuc, "%04o", bilgi.st_mode & 0777);
}

main() Fonksiyonu ve Argüman Ayrıştırma

main() fonksiyonu argv[1] parametresine bakarak hangi modda çalışılacağını belirler. -b modunda giriş dosyaları ve -o parametresi ayrıştırılır, -a modunda ise arşiv dosyası adı ve opsiyonel dizin parametresi alınır.

7. Sonuç

Bu projede tarsau adlı arşivleme programı başarıyla geliştirilmiştir. Program, ödevde belirtilen tüm gereksinimleri karşılamaktadır:

  • Metin dosyalarını tek bir .sau arşiv dosyasında birleştirebilmektedir
  • Arşiv dosyasını açarak dosyaları istenilen dizine çıkartabilmektedir
  • Dosya izinleri (okuma, yazma, çalıştırma) arşivleme sırasında kaydedilip açma sırasında geri yüklenmektedir
  • Hatalı giriş durumlarında uygun hata mesajları verilip program sorunsuz şekilde sonlanmaktadır
  • Binary dosyalar reddedilmektedir
  • En fazla 32 dosya ve 200 MB toplam boyut sınırı uygulanmaktadır
  • Makefile ile derlenebilmektedir

Geliştirme süreci GitHub üzerinden yürütülmüş olup proje deposuna https://github.com/YesilBaraN/tarsau adresinden erişilebilir.
